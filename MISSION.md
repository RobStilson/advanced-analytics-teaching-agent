# Mission

## Why I'm Learning This

I am a People Analytics practitioner who wants to move from ad-hoc model building to a **systematic, reproducible modeling workflow** using tidymodels in R. I already work with the tidyverse daily, but my ML work has been inconsistent — running individual models without a principled train/validate/test loop or a reusable pipeline.

## What I Will Be Able To Do

When this learning journey is complete, I will be able to:

1. **Build a complete modeling pipeline** from raw HR data to a tuned, validated model using the tidymodels framework
2. **Present model results** to HR leadership and business stakeholders — communicating predictions, uncertainty, and model limitations in plain language
3. **Apply the workflow to core People Analytics problems**: attrition/retention prediction, performance prediction, workforce planning/headcount forecasting, and survey/engagement analytics
4. **Explain and interrogate models** using tools like `{vip}` and `{DALEX}` so stakeholders can trust the outputs

## My Starting Point

- **R**: Intermediate — fluent in the tidyverse (dplyr, ggplot2, tidyr), writes own analyses
- **ML/Statistics**: Some hands-on experience — has run `lm()`, `glm()`, and similar models, but without a systematic train/validate/test workflow
- **People Analytics context**: Active practitioner working with HR data, producing analyses for business stakeholders

## Success Criteria

I can independently take a real People Analytics dataset (e.g. IBM HR Attrition), build a properly validated predictive model using tidymodels, and present the results — including model performance, key drivers, and predictions — to a non-technical HR audience.

## Primary Resource

*Tidy Modeling with R* by Max Kuhn & Julia Silge — https://www.tmwr.org/
